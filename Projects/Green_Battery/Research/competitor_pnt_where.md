## サービスの仕組み

```
衛星画像 → AI画像解析（空地・駐車場を自動検出）
                    ↓
          法務省登記APIと連携
                    ↓
     地図UI上で所有者情報・地番を表示
                    ↓
       アプローチ管理（CRM）で案件を追跡
```

## 主要機能

| 機能 | 内容 |
|---|---|
| 地図上検索 | 衛星画像から空地・更地を自動発見 |
| 登記情報取得 | 所有者氏名・住所・全部事項証明書を表示 |
| 公図重ね合わせ | 地図上に公図レイヤーを表示 |
| アプローチ管理 | 接触記録・進捗管理（CRM的機能） |
| SanSan連携 | 名刺交換済み企業をマークして識別 |
| インポート機能 | ユーザー保有データを取り込み・マッピング |
| 組織内メール共有 | チームで不動産情報を共有 |
| 海外対応 | 米国フロリダ・ラスベガス対応（PoC） |

## Green Battery が参考にできる点（優先度順）

### P1 — 登記情報の実連携
- 現在 `/api/registry` はモックデータを返している
- 法務省の登記情報提供サービスAPIと連携して実データを取得する実装が必要
- WHEREは「全部事項証明書の全情報」まで取得できる点を差別化に使っている

### P2 — 公図の重ね合わせ表示
- 地図UI上に公図（地番ポリゴン）を重ねて表示する機能
- 現在のLeaflet.jsダッシュボードに追加可能なレイヤー機能

### P3 — アプローチ管理（CRM）
- 候補地ごとに接触状況・メモ・進捗を記録する機能
- 不動産業者・事業者向けにはこれが実務上最も重要な機能

### P4 — 組織内共有
- チームで候補地リストを共有・コメントする機能
- SaaS的な多ユーザー対応
